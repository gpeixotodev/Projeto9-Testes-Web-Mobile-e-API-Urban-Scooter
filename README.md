Descrição do projeto

- Projeto integrador do Bootcamp de QA, reunindo os principais conhecimentos adquiridos ao longo da formação. Foram elaborados e executados casos de teste para a aplicação Urban Scooter, abrangendo testes funcionais na interface web, testes em dispositivos móveis e validação de APIs REST. Também foram identificados e registrados bugs no Jira durante a execução dos testes.

1. Objetivo do projeto

- Validar as funcionalidades da aplicação Urban Scooter em diferentes camadas (Web, Mobile e API), garantindo que os requisitos fossem atendidos e documentando as inconsistências encontradas durante os testes.

2. Resultado

- Foram desenvolvidos e executados casos de teste para validação de formulários, funcionalidades da aplicação mobile e endpoints da API. Os testes permitiram identificar falhas de validação, regras de negócio e inconsistências no comportamento da aplicação, que foram registradas no Jira para acompanhamento e correção.

3. Ferramentas utilizadas

- Microsoft Excel;
- Postman;
- APIs REST;
- JSON;
- Jira;
- Casos de Teste (Test Cases);
- Testes Funcionais Web e Mobile.

5. O que aprendi com esse projeto (habilidades/competências adquiridas)
- Planejamento e execução de testes em aplicações Web, Mobile e APIs;
- Elaboração de casos de teste utilizando classes de equivalência e valores-limite;
- Validação de respostas de APIs REST;
- Identificação, documentação e rastreamento de bugs no Jira;
- Aplicação integrada das principais técnicas de testes manuais.

6. Existem melhorias a serem feitas?

- Como evolução do projeto, seria interessante automatizar os principais cenários de teste utilizando Selenium e ferramentas de testes de API, ampliar a cobertura dos casos de teste e integrar a execução automatizada a pipelines de CI/CD para validações contínuas.

- Metodologia

- Análise dos requisitos da aplicação, elaboração dos casos de teste, execução das validações em ambiente Web, Mobile e API, comparação dos resultados obtidos com os esperados e registro dos defeitos identificados no Jira.

Como executar o projeto

- Executar os casos de teste definidos para cada módulo da aplicação (Web, Mobile e API). Para os testes de API, utilizar o Postman para enviar as requisições e validar os códigos de resposta e o conteúdo retornado. Registrar no Jira qualquer comportamento divergente identificado durante a execução dos testes.
